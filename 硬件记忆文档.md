# 硬件记忆文档

**用途**: 硬件参数、引脚定义、寄存器配置、协议规范等有效知识。错误排查看 DEBUG_CHECKLIST.md。

---

## 核心硬件参数

- **MCU**: 合宙 ESP32-C3 (Rev3, CH343), RISC-V 单核 160MHz
- **Flash**: 4MB, **DIO 模式, 40MHz**（不支持 QIO）
- **SRAM**: 400KB
- **音频模块**: 鹿小班 ES8311 + NS4150B
- **串口**: USB COM4, UART0 (GPIO20 RX, GPIO21 TX → CH343)
- **波特率**: boot ROM 115200 → 固件启动后 921600

## ES8311 I2C 寄存器配置（地址 0x18）

| 地址 | 名称 | 值 | 说明 |
|------|------|----|------|
| 0x00 | RESET | 0x40 | bit6=0 从机模式; bit4=1 释放复位 |
| 0x01 | CLK_MANAGER | 0xBF | **bit7=1 使用 BCLK 作为系统时钟**（ESP32-C3 无 MCLK） |
| 0x02 | CLK_DIV | 0x00 | 时钟分频 |
| 0x03 | ADC_CTL1 | 0x10 | ADC 输入音量 |
| 0x04 | ADC_CTL2 | 0x10 | ADC 增益 |
| 0x05 | ADC_CTL3 | 0x00 | ADC 混音器 |
| 0x06 | ADC_CTL4 | 0x00 | bit7=0 从机, bit6=0 上电 |
| 0x07 | DAC_CTL1 | 0x00 | 从机模式 |
| 0x08 | I2S_SEL | 0x10 | I2S 16-bit |
| 0x09 | MIXER | 0x0C | 混音器 |
| 0x0A | DAC_VOL | 0x0C | DAC 音量 |
| 0x0B | ADC_VOL | 0x0C | ADC 音量 |
| 0x0D | ANA_POWER | 0x0C | bit1=1 MICBIAS, bit0=1 ADC 上电 |
| 0x0E | GPIO_EN | 0x02 | GPIO 功能 |

### 寄存器初始化顺序
```
writeES8311(0x00, 0x40);  // 从机 + 释放复位
writeES8311(0x01, 0xBF);  // BCLK 时钟
writeES8311(0x08, 0x10);  // I2S 16-bit
writeES8311(0x09, 0x0C);  // 混音器
writeES8311(0x06, 0x00);  // ADC 从机 + 上电
writeES8311(0x07, 0x00);  // DAC 从机
writeES8311(0x0A, 0x0C);  // DAC 音量
writeES8311(0x0B, 0x0C);  // ADC 音量
writeES8311(0x0D, 0x0C);  // ADC + MICBIAS 上电
```

## I2S 配置参数

- **采样率**: 24000Hz（匹配 xiaozhi 固件）
- **位数**: 16-bit
- **格式**: I2S 标准（飞利浦格式）
- **模式**: Master（ESP32 产生 BCLK/WS）
- **TX slot**: I2S_STD_SLOT_LEFT（单声道）
- **RX slot**: I2S_STD_SLOT_BOTH（ADC 输出双声道）
- **BCLK 频率**: 24kHz × 2 × 16 = 768kHz

---

## 串口帧协议

### 格式
```
AA 55 [length:4 LE] [cmd:1] [data:length-1]
```
- 长度字段 = 1 + N（含 cmd 字节）
- 总帧长 = 7 + N 字节

### 下行命令（PC → ESP32）

| CMD | 值 | 数据 | 说明 |
|-----|----|------|------|
| REC_START | 0x01 | 无 | 开始录音 |
| REC_STOP | 0x02 | 无 | 停止录音 |
| PLAY_AUDIO | 0x04 | PCM 数据 | 播放音频 |
| PLAY_TONE | 0x06 | 无 | 播放 400Hz 测试音 |
| PLAY_MELODY | 0x07 | 无 | 播放小星星旋律 |
| READ_REGS | 0x08 | 无 | 读取 ES8311 寄存器 |
| DIAG | 0x09 | 无 | 音频诊断 |

### 上行帧（ESP32 → PC）

| CMD | 值 | 数据 | 说明 |
|-----|----|------|------|
| REC_DATA | 0x01 | PCM 数据 | 录音数据流 |
| REC_STOPPED | 0x02 | 无 | 录音停止通知 |
| REC_STARTED | 0x03 | 无 | 设备就绪 |
| PLAY_DONE | 0x05 | 无 | 播放完成 |
| REG_DATA | 0x08 | [reg:1][val:1] | 寄存器数据 |
| DIAG_DATA | 0x09 | [total:4][nonzero:4][ff:4] | 诊断数据 |

## WiFi 配置

- **STA 模式**: 自动连接保存的 SSID（Preferences 存储）
- **AP 模式**: SSID="ESP32-Voice", 网页 192.168.4.1 配置 WiFi
- **TCP**: port 8080，接收 PCM 数据
- **超时**: STA 连接 15 秒超时后切 AP 模式

---

## 音频链路

```
PC STT → PC LLM → PC TTS → PC resample(16→24kHz) → serial(921600)
  → ESP32 ringBuf → playTask → I2S TX → ES8311 DAC → NS4150B PA → 喇叭

麦克风 → ES8311 ADC → I2S RX → ESP32 → serial(921600) → PC
```

### 数据流关键参数
- **TTS 输出**: 16kHz / 16-bit PCM（PC 端用 np.interp 重采样到 24kHz）
- **串口传输**: 24kHz / 16-bit PCM（921600 波特率，约 384kbps PCM 数据）
- **播放缓冲**: 128KB 环形缓冲区
- **录音分片**: 每帧 512 样本（约 21ms）

## 小智固件（xiaozhi-esp32 v2.1.0）分析

已通过串口读取小智固件运行日志，确认以下信息：

### 关键发现
- **Board**: xmini-c3（与合宙ESP32-C3兼容）
- **ES8311 驱动**: 使用 `esp_codec_dev` 库（Espressif官方）
- **I2C 位址**: 0x18
- **ES8311 模式**: SLAVE（R00 bit6=0）
- **时钟源**: BCLK（R01=0xBF，bit7=1）
- **电源管理**: R0D=0x0C
- **串口波特率**: 115200（小智固件不切换波特率）
- **网络**: MQTT 连接云端服务器
- **音频编码**: Opus（16kHz → 重采样到24kHz）
- **唤醒词**: nihaoxiaozhi（小智你好）

### 小智 vs 我们的配置对比
| 参数 | 小智固件 | 我们的固件 | 差异 |
|------|---------|-----------|------|
| ES8311 模式 | SLAVE | SLAVE | ✅ 一致 |
| R01 时钟 | 0xBF (BCLK) | 0xBF | ✅ 一致 |
| R0D 电源 | 0x0C | 0x0C | ✅ 一致 |
| I2S 采样率 | 24000Hz | 24000Hz | ✅ 一致 |
| 初始化方式 | esp_codec_dev库 | 手动bit-bang I2C | ⚠️ 不同 |
| 串口波特率 | 115200 | 921600 | ❌ 不一致（开发调试用） |
| 通信协议 | MQTT | 自定义帧协议 | 架构不同 |

### 从 xiaozhi 学到的 ES8311 正确初始化顺序
1. 写 R01=0x30, R02=0x00, R03=0x10, R16=0x24, R04=0x10, R05=0x00（预配置时钟）
2. 写 R0B=0x00, R0C=0x00, R10=0x1F, R11=0x7F（预配置音量）
3. 写 R00=0x80（软复位+从机模式），延时50ms
4. 写 R00=0x80（释放复位，保持从机模式）
5. 写 R01=0xBF（BCLK时钟源）
6. 配置 R09/R0A（I2S格式16-bit，取消静音）
7. 配置 R08=0x10（I2S选择）
8. 配置其他系统寄存器
9. **最后写 R0D=0x0C**（电源管理：ADC+DAC通电）
10. 写 R10=0x1F, R11=0x7F（DAC音量）
11. 写 R0B=0x0C, R0C=0x0C（ADC音量）

## PC 助手功能需求（2026-06-12 更新）

### 核心功能
1. **文字输入工具** — 语音识别后，将文字直接输入到当前活动窗口（模拟键盘打字）
2. **网络搜索** — 通过 /search 前缀触发 DuckDuckGo 搜索
3. **时间查询** — 查询当前日期时间
4. **智能路由** — 用户问题自动判断处理方式：
   - 纯文字输入：简单命令/短语直接打字
   - LLM 参与：需要回答/推理的问题走本地 LLM
   - 网络搜索：含 /search 前缀或需要实时信息
   - 时间查询：含"时间""日期"等关键词
5. **本地大模型**：Qwen2.5-0.5B-Instruct（transformers 加载，CPU推理）
6. **语音输出**：TTS合成后通过ESP32播放
7. **WiFi配置**：参照 xiaozhi 格式（AP配网页 + STA模式）

### 数据流
```
用户说话 → ESP32录音 → 串口→ PC STT(SenseVoice) → 路由器判断
  ├─ 纯文字 → 键盘输入到活动窗口
  ├─ 搜索 → DuckDuckGo → 合成语音 → ESP32播放
  ├─ 时间 → datetime.now() → 合成语音 → ESP32播放
  └─ 问答 → LLM(Qwen2.5-0.5B) → 合成语音 → ESP32播放
```

### 命令格式
| 命令 | 示例 | 行为 |
|------|------|------|
| 无前缀文本 | "你好" | 自动判断：LLM参与 → TTS回复 |
| /ask | "/ask 今天天气" | 强制走LLM回答 |
| /search | "/search 巴黎" | 网络搜索 |
| /clear | "/clear" | 清空LLM对话历史 |

### WiFi 格式（参照 xiaozhi）
- AP模式：ESP32-Voice-XXXX（MAC后4位）
- 配网页：http://192.168.4.1/
- STA模式：自动连接保存的SSID
- TCP服务：port 8080（接收PCM数据）
- 超时：15秒连接失败切AP
